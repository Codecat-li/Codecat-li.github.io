vue解决跨域问题

使用vue-cli建立代理服务器

方式一：

缺点：不能灵活控制是否使用代理，不能配置多个代理

Vue.config.js文件

const { defineConfig } = require("@vue/cli-service");
module.exports = defineConfig({
  transpileDependencies: true,
  lintOnSave: false,
//配置
devServer:{
proxy:'http://localhost:服务器转发端口号'
}
});

方式二：

devServer:{
proxy:{
//'请求的前缀'
'/api':{
	//端口号
	target:'<url>',
    //重写路径，去掉请求前缀
    pathRewirte:{'^/api':''}
    //用于支持websocket
    ws:true,
    changeOrigin:true
	},
    '/foo':{
        target:'<other_url>'
        }
	}
}
配置后在端口号前面加上请求前缀
